CF915F Imbalance Value of a Tree

Posted on 2018-01-24 |

Link to problem
题目大意：给你一颗n个nodes的树，每个node有对应的weight， 求SUM(每两个不同nodes之间的path上最大的weight与最小的weight的差).
解析：刚看到这道题的时候想诶这个不是裸的树链剖分么,（孩子你还是太天真了），写完果断T了QAQ，后来看解答，觉得这道题可以说是思路很巧妙了。

蠢蠢的分界线

答案的组成可以看成是SUM(max) - SUM(min).
先来看求SUM(max), 首先把每个节点按照weight从小到大sort一遍，假设每个点一开始都是一个只包含自己的集合。 从最小的点开始(u)，遍历它所有的出边(u,v)，val(u) 就是当前所有经过u的path上的最大weight. 在计算有多少条经过u的path的时候有一个tricky的地方，就是维护一个当前u周围的点的数目sz，每当处理到一条新的已访问过的边(u,v’)的时候, 直接拿sz乘上 v’所在集合的大小就是新加入的path的数目。 最后不要忘了以u为端点的path，还有long long. 嗯，以上.
